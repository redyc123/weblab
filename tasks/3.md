Лабораторная работа №6

Реализация REST API в Laravel

# Подготовка окружения

Для выполнения лабораторной работы достаточно настроек окружения из предыдущей лабораторной работы.

Установите любое приложение, позволяющее тестировать REST API запросы. Например, [Postman](http://postman.com/)

# Выполнение задания

Используйте приложение из предыдущей лабораторной работы.

Реализуйте новый функционал:

- Установите [Laravel Passport](https://laravel.com/docs/8.x/passport#installation) для аутентификации пользователей при помощи OAuth2-токенов.
- Реализуйте генерацию токенов для каждого авторизованного пользователя. Пользователь должен авторизоваться через web-интерфейс и в профиле увидеть данные для OAuth2-аутентификации.
- Реализуйте методы для получения списка, записи, а также добавления и обновления данных при помощи REST API для основной сущности.
- Создайте [ресурс](https://laravel.com/docs/8.x/eloquent-resources) для вспомогательной сущности, добавьте данные основной сущности в вывод этого ресурса. Реализуйте GET, POST, PUT для связанной сущности.
- Протестируйте работу приложения. Web-версия приложения и API должны работать одновременно, новый функционал не должен нарушить работу предыдущей лабораторной работы

# Критерии оценки

**Базовый уровень:**

- Laravel Passport генерирует токены для каждого пользователя.
- Реализован метод GET для основной и вспомогательной сущности

**Расширенный уровень:**

- Критерии базового уровня
- Реализованы методы POST и PUT для основной и вспомогательной сущности
- При выводе в данные вспомогательной сущности добавляются данные основной.
- В данных REST отдельным полем передается признак "друга", если владелец записи является другом пользователя, по данным API которого выполняется операция.
